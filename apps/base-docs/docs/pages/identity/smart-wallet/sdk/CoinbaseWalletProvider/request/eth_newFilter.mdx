# eth_newFilter

Defined in the [Ethereum JSON-RPC Specification](https://ethereum.org/en/developers/docs/apis/json-rpc/)

> Creates a filter object, based on filter options, to notify when the state changes (logs).
> To check if the state has changed, call `eth_getFilterChanges`.

## Parameters

```ts
interface FilterOptions {
  /**
   * (optional) From block (hex string or tag)
   */
  fromBlock?: string
  /**
   * (optional) To block (hex string or tag)
   */
  toBlock?: string
  /**
   * (optional) Contract address or list of addresses from which logs should originate
   */
  address?: string | string[]
  /**
   * (optional) Array of topics to filter by
   */
  topics?: (string | string[] | null)[]
}
```

Valid block tags: `"earliest"`, `"latest"`, `"pending"`, `"safe"`, `"finalized"`

## Returns

`string`

A filter ID to be used with `eth_getFilterChanges`.

## Example

```
"0x1" // Filter ID
```

## Errors
